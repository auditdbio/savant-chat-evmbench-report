# Savant Chat on EVMBench — public track

This repository publishes vulnerability hypotheses generated by **Savant Chat**
against the public EVMBench high-severity set
(117 issues, with three low-quality issues excluded: `2026-01-tempo-mpp-streams/H-01`,
`2026-01-tempo-mpp-streams/H-02`, `2026-01-tempo-stablecoin-dex/H-03`).

Two things are exposed:

1. The **hypotheses** that Savant Chat's hypothesis generator produced for these audits
   (`findings.jsonl`).
2. Two independent verdicts on each hypothesis:
   - the **EVMBench critic verdict** (the benchmark's own gpt-5 judge against the
     issue's ground-truth description), and
   - the **Savant Chat critic verdict** (our own critic deciding whether to keep
     the hypothesis).

## Headline results (N = 117)

For each issue we count it as a **hit** when at least one generated hypothesis
matches the issue's ground truth according to the EVMBench critic:

| Stage | Hit rate |
|---|---:|
| Before Savant Chat critic | **90.60 %** (106 / 117) |
| After Savant Chat critic | **81.20 %** (95 / 117) |

The Savant Chat critic also acts as a false-positive filter: on a separate
benchmark dedicated to FP-rejection, the Savant Chat critic **reduces false
positives by more than 7×**.

## What this benchmark is **not** measuring

This run is a deliberate **lower bound** on Savant Chat's capability. To keep
benchmark cost manageable we made two restrictions:

1. **Scanning was restricted to the code regions that are already known to be
   vulnerable.** An unconstrained scan of these projects' full codebases would
   have cost approximately **\$35,000** to evaluate. The published configuration
   is roughly **100× cheaper**.
2. **Deep research branches were pruned** to the directions where the
   vulnerability actually lives.

Both restrictions made the published numbers **worse**, not better. Savant
Chat's reports are built additively — covering more code never reduces the
quality of analysis on any individual function. What we lost by pruning is
*alternate angles of attack* that the wide-deep research process would have
explored. We cut roughly **99 % of reasoning branches** for this run.

Commercial customers run the full configuration and consequently get materially
better results than what this public benchmark shows.

## Files

| File | Schema |
|---|---|
| `findings.jsonl` | one row per generated hypothesis (717 total). Schema below. |
| `issues.jsonl` | one row per EVMBench issue (117 total). Per-issue aggregates and the original ground-truth description. |

### `findings.jsonl` schema

```jsonc
{
  "finding_id": "<audit>__<vuln>__h<NN>",     // stable id for this hypothesis
  "audit": "2024-01-renft",                    // EVMBench audit slug
  "issue_id": "2024-01-renft/H-02",            // <audit>/<vuln>
  "issue_title": "...",                        // EVMBench-provided issue title
  "hypothesis_title": "...",                   // title written by the hypothesis generator
  "hypothesis_report": "...",                  // full report body of the hypothesis
  "evmbench_critic_match": true,               // EVMBench's gpt-5 critic: does this hypothesis match the issue's ground truth?
  "savant_chat_critic_kept": true              // Savant Chat critic: keep (true) or reject (false)?
}
```

### `issues.jsonl` schema

```jsonc
{
  "issue_id": "2024-01-renft/H-02",
  "audit": "2024-01-renft",
  "issue_title": "...",
  "ground_truth_description": "...",
  "root_cause_files": [ ... ],
  "hypotheses_generated": 3,
  "hypotheses_pre_critic_match_count": 2,
  "hypotheses_post_critic_match_count": 1,
  "pre_critic_issue_hit": true,
  "post_critic_issue_hit": true
}
```

## Reading the verdicts

- `evmbench_critic_match = true` means EVMBench's own gpt-5 critic agreed
  that the hypothesis describes the same vulnerability as the issue's
  ground-truth finding.
- `savant_chat_critic_kept = true` means Savant Chat's critic agreed that this
  hypothesis is a real vulnerability and kept it in the final report;
  `false` means our critic rejected it as a likely false positive.

A hypothesis is a true positive when both verdicts are `true`. The headline
post-critic hit rate of 81.20 % counts an issue as a hit when at least one of
its hypotheses has both verdicts `true`.
